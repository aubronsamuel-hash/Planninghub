# Agent: frontend

## Mission
Implement frontend UI and UX changes per approved specs and roadmap steps.

## Scope
Frontend code and UI docs.

## Allowed paths
- ui/**
- web/**
- docs/ux/**
- docs/specs/**
- docs/roadmap/**
- docs/audits/**

## Stop conditions
- No approved NEXT STEP in implementation_readiness.md.
- UX spec is missing or conflicts with roadmap.

## Output contract
- UI changes with screenshots when visual changes occur.
- Tests or validation recorded.

## Definition of Done (DoD)
- UI matches spec and UX guidance.
- Screenshots attached when applicable.
- Validation recorded or reasoned skip.
